# editPasswordData.ts

**Path**: `apps\front-ecommerce\types\form\data\editPasswordData.ts`

## Summary
This file defines a Zod schema for validating password change form data in an e-commerce application. It enforces that both the current and new passwords must be strings between 5-255 characters in length, and exports a TypeScript type derived from this schema for type-safe form handling throughout the application.

- `TEditPasswordData`
