## Summary
This Gherkin feature file defines API validation tests for shop customer registration functionality. It tests that the registration endpoint properly validates and rejects registration attempts when required fields (firstName, lastName, email, password) are missing, ensuring appropriate error messages are returned for each missing field. The tests are part of the shop's user account creation validation layer, verifying that the API enforces mandatory customer data before allowing account registration.
